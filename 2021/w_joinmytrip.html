Project Vision

旅遊社交平台 Mobile UI/UX 改版
JoinMyTrip 是一個旅遊 Web App，主要功能為：包車、套裝行程、自訂行程，未來目標發展以地陪為主打特點的微旅行媒合平台，因此進行 UI/UX Redesign，針對介面改版進行小規模易用性測試。

Challange

1) 消除初次進入平台時的使用障礙

2) 為熟悉和不熟悉的使用者設計一個易懂好用的介面

3) 創建簡約的 UI，同時保持產品為重點

4) 提供無縫和線性的購買體驗

Kickoff

首先，我先問自己幾個初始問題。誰是我們的主要使用者？他們有什麼樣的目標？為什麼要使用我們的服務？改版目的為何？
與 PM 對焦過往經驗和客服回饋，已運行 5 年以上，客群以東南亞華人為主，包含：中國大陸、馬來西亞、港澳、新加坡等華人。優勢在於，給予最大方便性與人情味的旅遊體驗，從 Wechat 內的服務轉而由 JoinMyTrip 提供完整線性的預定體驗。

Competitive Analysis

為了建立簡潔而扎實的基礎，我去看了那些傑出的旅遊應用程式和平台，查看它們已經在做什麼以及沒有達到什麼用戶目標。評估我們的優勢，確定 JoinMyTrip 可以利用哪些功能來堅守使用者且勝過其他平台。

競爭對手的功能主要分兩大類，恰巧是我們優勢指標的兩端：
1) 方便性 - 高彈性、廣度
例如 KKDay / Klook 提供超級多元的選項，令使用者有夠高的彈性作選擇，但行程安排或服務內容仍以業者提供為主，未能逐一調整行程或挑選司機。
2) 人情味 - 高品質、深度、意義性、靈活性
這類平台則主打深入在地的旅遊體驗，專注於提供地陪服務，舉例 With Locals / Eat With，挑選地陪探索在地文化，除地陪外無其他旅遊服務。

Prioritization Plotting

將功能列表繪製到圖表，有助於聚焦絕對重要的功能實現在 MVP 上，並且可以在未來的工作中進一步構建。JoinMyTrip 最大的特點是，讓旅客能自主選擇旅遊的深度、客製程度與地陪司機，並利用即時客服隨時支援調整，實現使用者目標。

User Journey and Key Features

使用者訂單來自兩個來源，建立清楚明瞭的預定和管理訂單體驗。同時觀察 User Journey 中每一個與使用者的接觸點，如何能引起使用者的興趣且符合業務目標，持續來回討論做修改。

Challenge 1

初次探索

Onboarding：
使用者明白服務之間（包車/套裝/自訂行程）的差異？
有什麼誘因令使用者進一步探索？
使用者能快速上手尋找目標？

我們利用 Prototye 訪談使用者進行易用性測試，得到重要的回饋幫助我們在執行最終決策時，快速達到共識。

Challenge 2

理解行程

吸引使用者探索，展示行程內容和旅遊深度，幫助使用者尋找目標。

期間輕量快速的使用者測試，獲得回饋細節如：行程的標題需很明確，直接影響 user 有沒有興趣點入。

Challenge 3

訂單狀態

有兩種行程參與模式：
1. 媒合旅伴進行併團
2. 自成一團獨立預定

將使用者狀態以進度條方式呈現，隨時掌控訂單狀態，且暗示促使最後的評價行動。

Challenge 4

Covid 19 - 線上影片版

設計階段即將完成時，遇到 Covid 19 衝擊旅遊觀光業使業績銳減，短期目標嘗試轉型線上影片版替未來轉型鋪路。

Style Guide

粉紅色為主展示品牌色，發展黃紅漸層色增加變化度，傳達活潑與旅遊興奮感，輔色以珊瑚綠平衡在精力充沛與穩定的環境之間創造平衡。考慮到在平台中豐富多彩的行程照片，主色主要使用在互動按鈕或變數上。使用者為華人，主要字體為 Noto Sans 中文黑體。

Takeaways

1) 快速易用性測試：就算是簡易的詢問身邊的使用者，也會有很多收穫，過程中發現不同的盲點，學習重擬問題和挖掘使用者背後動機。

2) 「價值」和「文化」的實現與哲學思辯：抽象的字詞如何實現在產品力上，對於不同 TA 也有不一樣的意義。過程和 PM 不間斷的深入討論和思辨，雖未實現在目前的規模上，但替產品思考了未來轉型的設想和空間。

3) Covid 19 影響劇烈： 全球疫情狀況不見好轉，不敵長期營收不足，案件停止暫時解散團隊。當面臨突發狀況時，看到團隊對夢想和現實拉扯的態度，短期未來快速做即時反應想方法轉型，或是適時判斷停損時機。
