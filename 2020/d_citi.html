花旗信用卡查詢服務 Chatbot

The project

銀行服務數位化是現代的趨勢，花旗銀行傳統複雜的系統之下，逐步將功能數位化。
此案以 AI 智能服務員對談的形式呈現，附於會員 APP 中，衍伸花旗信用卡查詢服務 Chatbot。

My role

  • 競品分析
  • UI Design
  • UI 模組化
  • UX Prototype
  • 客戶簡報提案
  • Develop 程式碼撰寫（html, css, jquery）
  • Work with PM & Design Assistant

Challenge

進行本案（信用卡查詢服務 Chatbot）之前，半年前已先進行過一次 Chatbot Beta 版稱為 eChat，局部功能先進行製作與測試，來回過程中，發覺客戶決策層級很多，決策時間也拉得較長、變數較多，因此和客戶溝通與交付時，面臨許多挑戰：

  • 屬於會員權限 APP 中的「全新附加功能」，難以實際深入產品，觀察研究其入口與產品其他全面功能。
  • Citi Brand Guideline 非常嚴格，UI 設計需細心嚴謹。
  • 客戶具裁決權的層級過多，交付之設計會反覆雕琢，難以定案。
  • 花旗內部現行系統年限已久且架構龐雜，運算速度與資料嫁接陷入困難。

Design Strategy

  • Prototype 動態提案

    面對多層決策者與設計抉擇困難的客戶，製作高完整度的A/B版 Prototype 進行提案，供客戶上乘回報與統整意見，協助專案執行效率與設計決策。

UX Usability

多種 UI & Prototype 嘗試與測試

版面配置嘗試了 2-3 版 UI 與 Icon ，包含大膽或基本的互動形式，製作 Prototype 供客戶與內部測試直覺性與可用性。

UI Design

資料精簡，重新定義權重

查找個人銀行資料時，很多資料都一起倒在版面上，沒有層級之分
——重新定義「重要且需要」的資訊，並將之作權重分配

  • Chat Strategy

    Chatbot 對話按鈕反饋

    按鈕若左右配置會有讓人二擇一的暗示性，評估與Survey後，為了鼓勵使用者嘗試更多選擇，統一使用「往下堆疊」的對話按鈕呈現方式。

  • 與客戶提案報告

    提案簡報

    製作視覺提案簡報，現場與客戶提案討論，分析設計理念、易用性與適切性、動態回饋。

    現場測試 Debug

    前端程式碼的測試站，也至現場和客戶討論動態問題與反饋細節。

Learning

高完整度 A/B 版 Mockup 製作 Prototype

在協助客戶對介面產生共識的過程非常順利。

前期花較多時間與功夫，促進彼此對畫面的共識與默契，後期再過其他畫面時，能直接判斷是否需兩版製作，對齊客戶期待，掌握專案節奏。

小樣本實際測試，進行迭代設計

以客戶為主的小樣本實測，進行介面上的迭代設計，半年前後的 eChat 和 Chatbot，從 menu、UI 介面到動態呈現，都有大小幅度的調整，每一次都針對資料結構的排列組合，測試多逐步完整。
